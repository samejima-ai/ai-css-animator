## 1. 規則結晶 vs パターン結晶（識別を鋭くする）

| | 規則結晶 | パターン結晶 |
|---|---|---|
| 発火 | 条件一致（binary, if-then） | 状況の**類似**（fuzzy, 「あの感じ」） |
| 転移 | 低（スコープ外で脆い） | **高（別対象へ転移する）** |
| 例（本セッション） | 「squash 後は rebase」「verifier 所有物は D5」 | ↓ PC-1 / PC-2 |

**判別テスト**: 「**異なる対象**に同じ定石が当たったか？」当たれば パターン結晶、単一条件にしか効かなければ 規則結晶。

## 2. 暫定スキーマ（pilot 版・開発者ペルソナの3機構を内蔵）

```yaml
pattern_crystal:
  id: PC-NNN
  発火条件:        # fuzzy な状況記述。embedding して現状況と cosine top-k で発火（機構1）
                  #   ※ OAuth/fine-tune 不可ゆえ HOT 常駐 + 索引近似。天井あり（重みに焼けない）
  反-発火条件:     # 「似てるが適用しない」状況を ≥1 個（機構3＝偽類推の防壁・検証可能）
                  #   発火時はまずここを照合し、一致したら抑制（falsifiable）
  力学:            # 競合する力（Alexander の forces）= なぜ自明でないか
  定石:            # 転移する応答スキーマ
  信頼度:          # 暫定 = ルールベース算出（機構2、統計学習はしない）:
                  #   firing_count（発火回数） × breadth（異なる対象型の数） × witnessed（高妥当性で目撃=true）
  source:          # cold:// 逆引き（不変条件#3）。pilot 段は本セッションを指す
```

**機構2の暫定方針（信頼度）**: 統計学習は不可（no-finetune）。よって `信頼度 = f(発火回数, 異なる対象型の広がり, 目撃の高妥当性)` のルールベース。high/medium/low の3値で粗く持つ（過剰な精度を装わない）。

## 3. 真 fuzzy サンプル（このセッションで発火を目撃したもの）

### PC-001: 自己言及的・早すぎる固化の罠 → 仮結晶＋二段階昇格

```yaml
id: PC-001
発火条件: >
  これから正典化/固定しようとしている対象が、それ自身「固定するな/可謬である」という
  性質や警告を内包していると感じるとき。「決める行為が、決める対象の本性に反する」感覚。
反-発火条件:
  - 対象が外形的・機械的で本性に暫定性を含まない（設定値の確定・純粋な事実記録・タイポ修正）
  - 既に十分な発火実績があり「拙速」と呼べない（熟した結晶）
力学: 「今 canonize する価値（設計軸の明確化・前進）」⇔「拙速固化のリスク（誤った正典の entrench・本性との矛盾）」
定石: >
  仮結晶（暫定・試用期間）として置き、現場発火の蓄積 → System2 審査（Council＋人間 P4）で
  本結晶へ二段階昇格。判断を先取りせず「判断する権利」を未来に残す。
信頼度: high   # 発火2回(theory thry01 / pattern-crystal-form pc1f01) × 異なる対象型2 × 目撃true。regime Council-gate にも同型
source: "cold://2026-05/this-session#thry01,pc1f01 reduction=DH"
```

→ **真のパターン結晶**: 「理論」と「パターン結晶の形式」という**別の対象**に同じ定石が転移した。fuzzy 認識（"これ、固定するなと言ってるものを固定しようとしてる感じ"）。

### PC-002: 高権威・境界に触れる決定 → 独立ゲートに献上（自分は判断を埋めない）

```yaml
id: PC-002
発火条件: >
  「この変更を、実装層の自分が単独で正典化していいのか？」という躊躇。高権威 artifact
  （philosophy / 境界規約 / 検証機構の所有物）や不可逆寄りの操作に『触れる感じ』。
反-発火条件:
  - 自分の mandate 内の可逆変更・明確仕様の素直実装・タイポ/フォーマット（過剰ゲートは Flow を止める）
  - 締切直前の軽微修正で献上コストが便益を上回る場合
力学: 「前進速度」⇔「権威の越境・独立性侵犯のリスク」
定石: >
  決定を独立ゲートへ献上する。canon/trade-off → Council、検証機構の所有物 → D5（人間）、
  不可逆 → 人間 P4。自分は判断（judgment）を提示するに留め、決定（final_decision）は埋めない。
信頼度: high   # 発火≥5回(decision3 P4 / mtbl01 / thry01 / pc1f01 / glossary D5) × 異なる対象型多数 × 目撃true
source: "cold://2026-05/this-session#decision3,mtbl01,thry01,pc1f01,glossary-d5 reduction=DH"
```

→ **真のパターン結晶**: 仕様正本化・規約・検証機構所有物・形式設計…と**多様な対象**に転移。fuzzy 認識（"これ、自分が独断で決めると越境する感じ"）。

### PC-003（境界例・参考）: squash 後の未 rebase 乖離 → reset --soft で純差分に削ぐ

```yaml
id: PC-003
発火条件: PR merge(特に squash)後の継続作業で、diff が「もう master にある内容」まで含んで膨らむ感じ
反-発火条件: [上流が fast-forward, 自分のブランチ未 push, 共有ブランチで他者がコミット中(force-push が破壊的)]
力学: 「履歴の clean さ」⇔「force-push の破壊性」
定石: git reset --soft origin/master → 純差分のみ単一コミット → --force-with-lease
信頼度: medium   # 発火2回だが対象型が単一(git PR) → 規則結晶寄り。パターン結晶としては境界例
source: "cold://2026-05/this-session#119to120,120to121 reduction=DH"
```

→ **境界例**: 発火はしたが対象型が「git PR」単一に近い＝規則結晶寄り。PC-001/002 との対比で「転移の広さ」が識別軸だと分かる good な反例サンプル。

## 4. pilot 計画（経営者＋哲学者の付帯条件）

- **期限**: 6 週 または 次の 3 cycle のいずれか早い方（経営者：寝かせ退行の防止）
- **終了条件（明示・哲学者の制度的慣性回避）**:
  - 成功 = 真のパターン結晶（転移実績あり）が **≥3 件**、暫定スキーマで表現でき、かつ **≥2 件が実際に別文脈で正しく発火**し、規則結晶の偽装でない → Council 再諮問で**本確定候補**
  - 失敗 = 上記未達、または「形式が暗黙知の言語化を**圧殺**した」観察 → スキーマ改訂 or 1 回だけ延長
  - **永続暫定の禁止**: 期限到達で必ず Council へ「本確定 / 改訂 / 棄却」のいずれかを諮る
- **観察目的（哲学者・最重要）**: 「形式の検証」ではなく **「形式が暗黙知の言語化を *促進* するか *圧殺* するか」の観察**。形式に当てはめようとして発火が痩せたら、それは形式の失敗シグナル
- **発効待ち**: 不変条件#8 は本確定後に発効（それまで「方針」）
